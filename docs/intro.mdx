# Projects

This site is a navigable documentation surface for my portfolio of projects, systems, and operational fronts.

It is designed to make project context durable and easy to revisit. Instead of reconstructing what each project is every time, this site keeps a stable description of groups and projects in one place.

## What is here

The main section is **Control Tower**, which contains:

- project groups
- project one-liners
- stable project context
- a navigable structure for revisiting active fronts

## How this site is built

This documentation is generated from a CSV snapshot that contains structured project context.

The current workflow is:

1. maintain the live project surface elsewhere
2. export or prepare a portfolio context CSV
3. generate Markdown documentation from that CSV
4. publish the result as a Docusaurus site

This keeps the publishing layer simple and makes it easy to rebuild the site from a clean snapshot.

## Main entry point

Go to [Control Tower](/control-tower) to browse the current project context.

## Scope

This site is not meant to be a full activity log, a journal, or a task board.

Its job is narrower:

- preserve stable context
- reduce re-explanation
- support weekly sweep and review
- provide a clean surface for project navigation

## Status

This is a lightweight generated surface and will evolve over time as the portfolio structure, grouping, and context fields improve.
