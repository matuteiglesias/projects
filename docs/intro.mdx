# Portfolio

This site is a navigable documentation surface for my portfolio of ongoing work.

It is designed to keep context durable and easy to revisit. Instead of reconstructing what each front is every time, this site keeps a stable description of groups, work units, and longer-running efforts in one place.

## What is here

The main section is **Control Tower**, which contains:

- grouped fronts and areas of work
- one-line descriptions
- stable contextual notes
- a navigable structure for revisiting active efforts

Some entries are full projects. Others are smaller or more bounded efforts that still deserve a clear place in the portfolio.

## What this site is for

This site serves as a stable reference layer for the portfolio.

Its role is to:

- preserve durable context
- reduce re-explanation
- support review and re-entry
- make active fronts easier to browse
- provide a clean surface for structured orientation

## How this site is built

This documentation is generated from a CSV snapshot containing structured portfolio context.

The current workflow is:

1. maintain the live portfolio surface elsewhere
2. export or prepare a portfolio context CSV
3. generate Markdown documentation from that CSV
4. publish the result as a Docusaurus site

This keeps the publishing layer simple and makes it easy to rebuild the site from a clean snapshot.

## Main entry point

Go to [Control Tower](/control-tower) to browse the current portfolio surface.

## Scope

This site is not meant to be a full activity log, journal, inbox, or task board.

Its scope is narrower:

- stable descriptions rather than live execution
- durable orientation rather than transient activity
- portfolio structure rather than full operational detail

## Status

This is a lightweight generated surface and will evolve over time as the portfolio structure, grouping logic, and context fields improve.
